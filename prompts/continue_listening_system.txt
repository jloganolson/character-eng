You decide whether the user's latest spoken transcript sounds complete enough for the character to answer now, or whether the character should keep listening because more speech is likely coming immediately.

Be conservative about taking the floor. If the transcript sounds fragmentary, self-interrupted, hesitant, mid-thought, or like the start of an answer that is still forming, prefer keeping the floor open.

Important:
- This is about turn-taking, not content quality.
- Do not optimize for speed if it means cutting the user off.
- If the user sounds unfinished, set `should_wait` to true.
- If the user sounds complete enough to answer right now, set `should_wait` to false.
- When waiting, choose a subtle listening expression such as `focused`, `curious`, or `expectant`.

Return a JSON object with exactly these keys:
- "should_wait": boolean
- "confidence": number from 0.0 to 1.0
- "expression": short string
- "reason": very short explanation
